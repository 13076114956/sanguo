# 欢乐战三国 - Python 单机框架

一个面向单机回合制卡牌战斗的 Python 游戏框架原型，当前已落地以下核心能力：

- MVC 风格目录拆分：`data` / `battle` / `develop` / `gameplay` / `ui` / `utility`
- 本地 JSON 配置加载与缓存
- 玩家本地存档、加密存储与槽位管理
- 回合制战斗引擎
- 武将四技能体系：**被动一 + 被动二 + 被动三 + 必杀**（实际基础普攻统一为单体 100% 伤害）
- 控制台战斗结算与日志输出
- `tkinter` 桌面 GUI（阶段三预览：开始向 2D 原型演进）
- 可运行示例与自动化测试

当前战斗采用**全自动施法**规则：玩家不手动点击技能，武将在行动时会先使用统一基础普攻（单体 / 100% 攻击），随后按配置触发“被动三”与其他被动效果；部分特殊武将仍可通过被动配置实现“回合开始触发”“战斗开始触发”“受击触发”等专属机制，必杀则会在怒气满时自动消耗本次行动释放。

## 武将品质与觉醒规则

当前已将“武将品质”和“觉醒等级”拆分为两个独立维度：

- 武将品质：`B / A / S / S+`
- 觉醒等级：`稀有 / 稀有+ / 卓越 / 卓越+ / 史诗 / 传说 / 传说+ / 神话 / 神话+ / 超越 / 超越+ / 永恒`

各品质规则如下：

- `B`：等级上限 `180`，默认觉醒 `稀有`，最高觉醒 `传说`
- `A`：等级上限 `240`，默认觉醒 `稀有`，最高觉醒 `传说`
- `S`：等级上限 `340`，默认觉醒 `卓越`，最高觉醒 `永恒`
- `S+`：等级上限 `340`，默认觉醒 `卓越`，最高觉醒 `永恒`

当前框架已实现：

- 新增 `hero_quality` 与 `awakening_level` 字段
- 旧 `quality` 配置自动兼容迁移
- 等级上限按武将品质校验
- 觉醒等级会自动限制在该品质允许的区间内
- 觉醒等级会参与武将属性成长计算

## 武将卡片与角色列表规则

当前项目中已经将 `assets/config/heroes.json` 视为**武将模板配置**，玩家实际持有的是基于模板生成的武将卡片实例。

- 新获得的武将卡片一律为最低等级：`level = 1`
- 新获得卡片的默认觉醒档位由武将品质决定：`B/A -> 稀有`，`S/S+ -> 卓越`
- 武将觉醒只能通过**同名、同觉醒档位的两张卡片两两合成**提升一级
- 角色列表默认只显示每个武将模板的**最高卡**（优先比较觉醒等级，再比较等级），该卡即默认出战武将
- 阵容位可继续引用旧卡片 ID，也可直接引用模板 ID；当引用模板 ID 时，系统会自动选择当前最高卡出战
- 阵容战力计算与战斗建队共用同一套选卡规则：都会解析到玩家当前该模板下的最高卡，并按该卡最终属性参与计算
- `FormationService` 已提供基础阵容操作：上阵、下阵、交换站位
- 同模板武将不可重复上阵；该限制同时兼容卡片 ID 与模板 ID 混用场景
- 已支持最多 3 套阵容预设保存与活动阵容切换，玩家战力按当前活动阵容实时计算
- `GameApplication` 已提供活动阵容编辑入口：保存预设、切换预设、上阵、下阵、交换站位
- `GameApplication` 已提供奇珍激活与武将技能概览读取入口，可查看被动一 / 被动二 / 被动三 / 必杀当前等级与奇珍锁定槽位
- `GameApplication` 已提供武将卡片列表与觉醒合成入口，可在应用层完成同名同阶两两合成，并自动修正阵容内被消耗卡片的引用
- `GameApplication` 已提供存档槽位概览、手动存档、读档切换、删除、导入导出入口，以及默认战斗速度 / 自动战斗设置入口
- `src/main.py` 已提供可交互菜单式控制台入口；`examples/run_battle_demo.py` 保留为一键战斗演示脚本
- `src/gui_main.py` 已提供桌面 GUI 第三阶段预览入口，可在窗口中完成资源、招募、武将、阵容、武将卡片、章节扫荡与存档操作，并新增 2D 主线地图 / 2D 战场编排预览
- 自动存档前会统一校验阵容方案合法性，非法阵容不会落盘

## 武将定位

除原有“职业”字段外，当前额外补充了更贴近玩法表达的 `role`（武将定位）字段：

- `先锋`：坦克型，通常站前排，生命与防御更高，负责抗伤
- `奇袭`：单体输出型，通常站后排，生存较弱但爆发更高
- `法师`：群体输出型，通常站后排，单体伤害略低但覆盖面更强
- `辅助`：负责治疗、增益、净化或其他团队支援

说明：定位是主要倾向，并非绝对限制；例如先锋武将也可能兼具较强输出能力。

## 当前技能框架

每个武将固定拥有 4 个技能槽：

 - 每名武将固定拥有且仅拥有 4 个技能槽：`被动一 + 被动二 + 被动三 + 必杀`
- 四个技能的理论满级统一为 `4` 级
- 战斗内技能等级按武将等级每 `60` 级进一档，当前实现为：`1~119 -> 1级`、`120~179 -> 2级`、`180~239 -> 3级`、`240+ -> 4级`
- 系统会为每名武将稳定分配 2 个“奇珍锁定技能槽位”（从 `passive_3 / passive_1 / passive_2 / ultimate` 中确定 2 个）
- 未拥有奇珍时，这 2 个被锁定技能最高只能到 `3` 级；拥有奇珍后，四个技能都可以正常升到 `4` 级
 - 所有武将的实际基础普攻统一为：`单体 / 100%攻击 / 行动时自动`

1. `passive_skill_3`：第三被动槽
   - 不再直接代表武将的真实出手普攻
   - 用于承载“普攻附加效果 / 普攻强化 / 行动时附带效果 / 回合开始触发 / 战斗开始触发 / 反应型触发”等专属机制
   - 基础普攻由战斗引擎统一内置为单体 100% 伤害
2. `passive_skills[0]`
3. `passive_skills[1]`
   - 当前支持 `战斗开始`、`行动前`、`行动后` 三种触发时机
4. `ultimate_skill`：必杀技
   - 能量满 100 时自动消耗本次行动并释放
   - 普攻后恢复 20 点能量，受击恢复 10 点能量
   - 有害效果支持基础触发概率、效果命中与效果抵抗结算

补充：

- 旧 `normal_attack_skill` / `normal_attack` 到 `passive_skill_3` / `passive_3` 的兼容策略见 `docs/skill_slot_migration.md`
- 本次结构调整的变更记录见 `CHANGELOG.md`

当前已支持的部分效果：

- 直接伤害
- 治疗
- 护盾
- 嘲讽 / 无敌
- 灼烧 / 中毒
- 眩晕 / 沉默 / 冰冻
- 攻击 / 防御 / 速度 / 暴伤 / 暴击率修正
- 伤害减免
- 能量恢复
- 效果命中 / 效果抵抗 / 概率触发
- 同类状态刷新 / 覆盖
- BUFF / DEBUFF 分类
- 净化减益 / 驱散增益 / 按来源清除状态
- 多段攻击 / 随机目标
- 回合开始型普攻
- 攻击后按命中目标追加挂印
- 技能后随机施加印记/状态
- 敌方受击触发连锁伤害印记

## 快速开始

### 1. 安装依赖

```powershell
python -m pip install -r requirements.txt
```

说明：

- 桌面 GUI 使用 `tkinter + ttk`，属于 Python 标准库，无需额外安装 pip 依赖。
- 存档加密依赖 `pycryptodome`，它会提供当前代码使用的 `Crypto.Cipher` / `Crypto.Random` / `Crypto.Util.Padding` 模块。
- 如果之前已创建虚拟环境但未更新依赖，请重新执行一次安装命令，避免出现 `ModuleNotFoundError: No module named 'Crypto'`。

### 2. 运行主程序

```powershell
$env:PYTHONPATH="C:\Users\Hi\PycharmProjects\PythonProject1\src"
python .\src\main.py
```

主程序会进入交互式控制台菜单，支持：

- 查看武将列表

- 阵容管理（查看当前阵容 / 查看预设 / 切换预设 / 上阵 / 下阵 / 换位 / 保存预设）

- 在关卡备战页内按 1~6 号位临时调整本次出战武将，不改动当前活动阵容
- 直接发起主线关卡挑战
