# 欢乐战三国 GUI 路线图

> 更新时间：2026-05-21

## 1. 当前结论

当前项目原本只有菜单式控制台入口，现已开始引入 **桌面 GUI**。

首选技术方案：**`tkinter + ttk`**

选择原因：

- Python 标准库自带，**无需新增运行时依赖**
- Windows 环境可直接使用
- 足够承载当前项目以“列表 / 详情 / 日志 / 按钮操作”为主的原型交互
- 可以直接复用 `GameApplication` 作为统一业务入口，避免再写一套玩法逻辑

## 2. GUI 分阶段计划

### 阶段一：最小可运行桌面壳（已开始实现）

目标：让项目不再只有控制台，可以通过窗口进行核心只读浏览与部分关键操作。

本阶段范围：

- 主窗口与标签页导航
- 资源与招募页
- 武将列表 / 武将详情页
- 主线关卡列表 / 关卡备战页
- GUI 内直接触发部分关键玩法操作

当前已落地的 GUI 能力：

- 查看资源状态
- 领取挂机收益
- 快速挂机
- 购买体力
- 单抽 / 十连元宝招募
- 查看武将列表与详情
- 武将升级 +1 / 尽可能升级
- 激活奇珍
- 查看主线关卡列表
- 查看关卡备战信息
- 直接开始关卡战斗
- 扫荡单个已通关关卡
- GUI 消息日志展示

对应入口：`src/gui_main.py`

### 阶段二：核心操作补齐（已完成本轮主功能）

目标：把已经在控制台中可用的关键管理能力逐步迁移到 GUI。

本轮已补充：

- 阵容管理
  - 查看当前阵容
  - 上阵 / 下阵 / 换位
  - 预设切换与保存
- 存档管理
  - 槽位概览
  - 手动存档
  - 切换读档
- 主线强化
  - 章节一键扫荡
  - 章节筛选与推荐战力提示
- 武将管理
  - 武将卡片列表
  - 觉醒合成

### 阶段三：向 2D 原型过渡（本轮已开始）

目标：让 GUI 从“功能壳”向“2D 原型玩法界面”过渡。

本轮已落地：

- 新增 `2D 场景` 页签
- 基于 `tk.Canvas` 的章节节点地图
- 基于 `tk.Canvas` 的 6v6 战场布阵预览
- 从地图节点点击同步选择关卡
- 战斗后在 2D 场景页展示胜负结果摘要
- 基于战斗日志的逐帧回放时间线
- 回放控件：上一帧 / 下一帧 / 播放暂停 / 进度条 / 速度切换
- 当前行动单位高亮、目标受击/治疗边框、槽位飘字反馈
- 最小补间动画：飘字上浮、行动闪光、受击抖动
- 已补充回放友好的结构化战斗日志元数据（单位 ID / 阵营 / 生命变化 / 能量变化 / 状态变化）

后续继续补充：

- 战斗过程逐帧回放、受击飘字、技能闪光
- 更细粒度的战斗日志元数据（group_id / sequence / 多段命中链路）
- 页面内消息提示与弹窗统一化
- 更清晰的数据卡片式展示
- 可选的页面路由与状态同步机制
- 章节大地图 / TileMap / 资源贴图缓存
- 为后续更复杂 GUI（如 PySide6 / Godot / pygame）预留迁移边界

## 3. 与现有架构的关系

GUI 层原则：

- **不直接改业务数据**
- 所有玩法操作统一调用 `GameApplication`
- GUI 只负责：
  - 读取应用层数据
  - 触发应用层动作
  - 展示结果与消息

这样可以保持：

- 控制台版和 GUI 版共享同一套业务逻辑
- 自动化测试主要继续覆盖应用层与战斗层
- GUI 渐进演进时不容易破坏现有闭环

## 4. 如何运行 GUI

在项目根目录执行：


如果使用 PyCharm，也可以直接运行：

- `src/gui_main.py`
